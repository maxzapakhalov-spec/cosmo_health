# **1. Назначение документа**
В отчете описан разработанный прототип автономной системы поддержки здоровья экипажа, его архитектура и принципы работы. В документе приведены описание архитектуры, используемых технологий и алгоритма анализа, а также инструкция по запуску и демонстрации прототипа.
# **2. Архитектура решения**
Прототип реализован на Python и запускается как настольное приложение. Программа формирует запрос к внешнему LLM-сервису и отображает полученный результат в интерфейсе. (DeepSeek Chat Completions API) и визуализирует ответ.
## **2.1 Компоненты**

|**Компонент**|**Назначение**|
| :- | :- |
|Графический интерфейс (GUI)|Форма ввода показателей (пульс, HRV, SpO2, давление, температура) и текстовое поле самочувствия; зона вывода результата; индикатор загрузки.|

|Обработка ошибок|Проверка наличия API-ключа, обработка HTTP-ошибок и исключений парсинга.|

# **3. Стек технологий**
## **3.1 Язык и среда выполнения**
Python 3.x (желательно Python 3.11). Интерфейс приложения формируется с использованием библиотеки Flet и запускается вместе с основной программой.
## **3.2 Внешние сервисы**
Для анализа данных используется облачный API DeepSeek (метод chat/completions).Запрос отправляется на endpoint https://api.deepseek.com/chat/completions с авторизацией по Bearer API key.
## **3.3 Форматы данных**
Обмен данными с ИИ-сервисом выполняется в JSON. Приложение требует от модели ответ строго следующей структуры:

|{<br>`  `"recommendations": ["..."],<br>`  `"conditions": [<br>`    `{"name": "...", "probability": 50}<br>`  `]<br>}|
| :- |

# **4. Использованные алгоритмы и модели ИИ**
## **4.1 Модель**
В программе применяется Deepseek AI через публичный API DeepSeek. Модель используется для анализа введённых показателей и формирования структурированного ответа в заданном формате.
## **4.2 Алгоритм анализа в прототипе**
1. Сбор пользовательских параметров: пульс, HRV, SpO2, давление, температура, свободное описание самочувствия.
1. На основе введённых параметров формируется текст запроса к модели с указанием требуемого формата ответа (JSON).
1. Отправка запроса в DeepSeek API (chat/completions) и получение ответа модели.
1. Полученный ответ проверяется и преобразуется из JSON в объекты программы, после чего результаты отображаются в интерфейсе.
# **5. Описание реализации прототипа (по исходному коду)**
## **5.1 UI-слой**
Интерфейс построен на Flet и состоит из набора «карточек» (Container) с полями ввода и области вывода результатов. Для обеспечения единого стиля интерфейса используются заранее заданные цветовые константы и вспомогательные функции для создания элементов формы label(), input\_field() и card().

## **5.3 Запуск**
Скачайте ехе файл и пдф документ в одну папку и дважды кликните по скачанному ехе файлу

После запуска откроется окно приложения. Введите показатели и нажмите кнопку «Анализ».
## **5.4 Демонстрационный сценарий**
Для демонстрации работы системы целесообразно заранее подготовить 2–3 набора показателей, соответствующих различным состояниям.

Пример:

|**Диагноз**|**Пульс**|**HRV**|**SpO2**|**Давление**|**Температура**|
| :- | :- | :- | :- | :- | :- |
|Норма|70|60|98|120/80|36\.6|
|Температура|95|35|97|130/85|38\.5|
|Гипоксия|110|25|90|120/80|37\.0|

В поле «Самочувствие» добавьте короткое описание симптомов (например, «озноб, ломота, кашель»).
